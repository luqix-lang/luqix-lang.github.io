Basics try-statement

This is used to catch errors in code.

The try statement allows you to define a block of code to be tested for errors while it is being executed.

All we do is demacate some code for the try statement and if an error occurs, then it will be returned as an exception, that is passed to the except statement that should be defined right after the accompanying try statement as the variable specified.

fn sum(a, b) return a + b try { sum(7) // an error must araise since we given in one argument } except (e) { print(e) // this is our error as an enum data }

This is not that safe to use since the interpreter sections keep changing and so it won't work perfectly and mostly the Exception object.

Specialities

No new scope is created, so the break and return statements will work here. Next know that we can define a free try statement without a corresponding except statement incase we don't wanna catch the error.

try { // try some code }

This can be used incase you don't want to catch the error and just bypass it.
