Datatypes

Luqix has a few datatypes since its interpreted, but i think they can suit to carry all data required in your code and they only total to 7.

You are also capable of creating your own datatypes called objects from the existing datatypes to which you provide methods for their manuplation. coming soon example.

You may find others provided as simple libraries either written in native code or built on top of luqix classes.

Info
First to check the type of the data use the type(...) function or find data length using len(...). All these are from locals module. Good enough all its functions are already imported to the working scope.

# number

The number datatype includes all kinds of numbers in programming as due to weak typing, the interpreter will get a version of what it wants from your given code.

intergers, float, double, real. byte, short -> number

Did you know?
The number is a double in native code that is casted to the required number when needed.

a = 12345 // integer b = 34.5 // float c = -475 // hegative money = 1_000_000.5

Must know!
All arithmetic operations work with numbers so if used on another datatype, it will be seen as a zero, in such cases, each datatype as a corresponding module to handle such operating. Checkout the math module

# string

The string is a bunch of characters both letters, numbers and marks. This will be created for any character inside quotes, '...' "...", read from a file or decode bytes.

name = 'luqix programming language' user = prompt('Enter name: ') ascii = 'abcdefghijklmnopqrstuvwxyz0123456789!@$%^&*(())-+{}\/..' hex = '\x72' // r

Note
For hex-codes and other special string encodings be very kin as due to poor usage, you may get an error that is hard to resolve now since error handling is still poor.

Formatted string

This one makes life easy as at parsing time the string is well formatted to special code that binds all assigned code in curl braces as one. That escapes numerous string concations.

a = 4 b = 4 print(f'And the total is {a + b}.') // similar approached c = str.concat('And the total is ', a + b, '.') print(c)

Raw string

Enables you capture a string in its raw form minus un-decoding its special characters and string encodings.

a = 4 b = 4 print(f'And the total is {a + b}.') // similar approached c = str.concat('And the total is ', a + b, '.') print(c)

# list

This is an array capable of storing multiple datatypes and to which other operations can be done like indexing, duplicating and many more.

Note
The list's ability to store multiple datatypes has no effect to efficience as in backend, all luqix datatypes are the same object type.

Info
For the iterator cases we are not yet there but methods like indexing are provided and a local module object, list just to manuplate and use lists data.

list object is automatically imported to the runtime when the program starts and its the one to handle lists data other than each list object having its own attributes. This is done for memory optimization.

items = ['a', 'b', 'c', 1, 2, [3, 4]] print(attr(lists)) // ['append', 'clear', 'contains', 'copy', 'ext...

We use [...] to make lists but their is an option too using (...) although this has a trick. As for code readability, we use the python tuple idea. If such (true) with one item, its seen as a math bracket but if multiple members (true, 1) or a trailing , (1,) then its a list.

tuple_like = (1, 2, 'hello', 4, 'world') // [1, 2, 'hello... x = (1) // 1 y = (1,) // 2 z = (true, 1) // [true, 1]
wild = ['lion', 'rat', 'hare', 'hyena'] dom = ['cat', 'dog', 'racoon'] animals = list.join(wild, dom) print(animals) str.sort(animals) print(animals) animals[1] // dog

Must know?
builtin sorting algorithms are not yet there but all is to be ready in the stable version. You can sort this data with native luqix too.

# dictionary

Abbreviated as dict, Dict data is inspired from already existing languages, its a javascript object, python dictionary with almost no change. Data is handled the same way and a clue for its backend is that it is a real associative array in native code with the a string as a key and a luqix datatype as the value.

This minimizes on the level of wrapping but satisfy dynamicness during code interpretation to machine code.

Note
The dict module has the functions to handle dictionary data but remember its already imported in the working scope.

json module is a builtin module which when combined with the dict datatype, you can communicate with many systems including the tremendous Javascript and its unique web capabilities.

data = {a: 'A', b: 'B', c: 'C', d: 123} print(data['c']) // C

For dynamic purposes you must use strings as keys.

settings = { 'theme': 'dark', 'font size': 14, 'open files': ['main.lqs', 'wav.mp3'] }

Getting items and manuplating dict data.

print(attrs(dict)) // ['clear', 'copy', 'get', 'has', 'keys', 'pop... data[a] = 'abcd' print(data) // { c: 'C', a: 'abcd', b: 'B', d: 123 } copy_of_items = dict.copy(data) keys = dict.keys(data) // ['c', 'a', 'b', 'd'] values = dict.values(data) // ['C', 'abcd', 'B', 123]

Must know
Don't expect the right order for inserting the data as dict capabilities don't mind the order and its done for optimisation purposes.

This datatype would make a great app settings store for revisiting and updating and if joined with json module, the program would come to life. Checkout an example

# boolean

These are only two and used for inquality expression. The true and false keywords are used to make them. They are equvalent to 1 and 0 but best for readability.

Note
The list's ability to store multiple datatypes has no effect to efficience as in backend, all luqix datatypes are the same object type.

x = true y = false

To view there number format we use the numb object one as the argument.

numb(true) // 1 numb(false) // 0

Booleans are over saturated as runtime return values as many functions default to returning them.

file = require('file') sample = 'app.lqs' bool = file.exists(sample) // expect true or false if (bool) print(file.isfile(sample)

Must know
Using true or 1 or false or 0 has not effect anythin since the runtime see only the number value.

# none

This is simply null in machine code as it represents no data, its fit for empty arguments as its utilizes no memory at machine level.

n = none

Its number value is 0 just for dynamicness but behind the hood its no data.

numb(none) // 0

Must know
Functions to which have no return statement return a none too.
