sys module

Module sys

This builtin module provides variables that control the interpreter and functions to interact with the interpreter or change its behaviour.

The variables in here always have a default value so endevour to replace with a relevant data type or else the weak typing will save you in a weird way.

# path()

Returns the paths used to locate luqix modules, you can append, do all list functions to it and it will be changed in realtime globally but please mind not to change the datatype. Or else the interpreter won't find your modules.

n = sys.path() // [...]

# argv()

Returns the a list of the command line arguments passed in as the program is being started.

As usual the first item is the filename calling the program or accordingly if using the other luiqx relevant commands.

x = sys.argv() // [...]

# executable

Returns a string path of where the interpreter executable or binary file is located. This could be handy incases of modulizing a massive project.

p = sys.executable

# modules_path

Returns a dictionary or the modules that have been imported in the working scopes globally.

mp = sys.module_paths

# getvar(name)

Returns a value held by the variable as named in the argument provided as a string.

Takes only one argument and its a must to be given.

name = 'Cutie' print(sys.getvar('name') // Cutie

# setvar(name, value)

Adds a new variable to the calling scope or even it just replaces its value with another if it exists.

Takes only two arguments a string name of the variables and then a value of any data-type.

sys.setvar('phone', 'iPhone') print(phone) // iPhone

# delvar(name)

Deletes a variables from the calling scope out of the interpreter runtime and gargabe collector.

delvar('len')
