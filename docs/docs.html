Introduction

Hello everyone, the Luqix team welcomes you to our official website documentation where we talk about the language in general, its usage, bugs and capabilities and also learn the official writing of the new luqix programming language.

Glad to say that its written in dlang.

Know about Luqix

  • Luqix is object oriented and tries all its data somehow as objects.
  • It has highly customizable syntax thanks to the multi code demacation options inbuilt its scratch crafted lexer and parser.
  • Although its data is all objects and so it has attributes. Each data having an individual attribute is expensive in memory issues and so functions are provided separately to reuse them to manuplate different data so as to save memory.
  • There is hope for dynamic file loading.
  • Speed is a proirity although not yet archived, hope you could bare.
  • Error handling sofar is not at its best but you could use the existing as updates come in. The try statement archives that.

Goals

  • We aim for luqix to be a simpler entry coding language for completely new programmers. Having the same concepts with mature languages and yet presenting them in there easy formats.
  • Just another alternative to the existing programming languages.
  • No gap is claimed to be filled but all needed is coding and deploying easily.
