Basics and syntax

Luqix code is written in the C way or indentive way, thats to say you can use alot of whitespaces in your code or non at all. The indentive code may not be very safe now as bugs may not be visible to the parser but if you write it properly you are on a safe side or why not rely on the C way.

Variables

Luqix doesn't have things like immutable and constant variables since one's code implementation is enough to have these in the code, never the less if a super need comes who knows.

There exists no global variables but using lists tricks and Objects you can achieve that, the options are endless. The let keyword is just for code redability and can be used where you want because it doesn't reach the parsing stage.

// all these are variables program = 'hello world!' let file = require('file') x = 9; y = 8 z = (x, y)

Variables don't have types, they can handle any a datatype which changes throughtout the execution.
Scoped variables like in functions are not available outside their scope.

fn foo(){ a = 'abcdef' } //ReferenceError 'a' is not defined, --- in this scope. print(a)

Functions

As usual, luqix functions are blocks of reusable and organised code that usually perform a single or related action. They may require arguments if assigned parameters and a name that is used to make use of it in the code.

Functions return data after performing some task if assigned to do so by the return keyword or it defaults to none datatype.

fn foo(a, b) { print(a, b) return 'I like ice cream.' }

Calling functions

To call a function you simply use the parenthesis and supply it with the arguments required by the function separated by commas.

Warning !
Please give the function call the right number of arguments to avoid errors, these will be harder to handle for now.

ret = foo(3, 7) // 3 7 print(ret) // I like ice cream

Default parameters

Functions can have default parameters assigned during the writing of the function that are over written if you want during the function call.

This may not be useful now as its buggy and unfinished but expect the best out of default function parameters in stable releases.

fn hello(name='Joy'){ print(f'hello {name}') } // both are okay hello('Alice') // hello Alice hello() // hello Joy

Default function parameters are not safe for use, atleast for now.

Nameless functions

Similar to arrow functions in javascript luqix provides almost the same function feature as you can easily define a function and yet assigning it as data for a certain variable or argument.

The syntax is friendly i hope. ? informs the parser about the awareness of the nameless function and the rest happens like a normal function defination.

setTheme = ?(theme){ // .... } setTheme('light')

If statment

If statements are always the same in may programming languages, taking the if keyword accompanied by elif and else keyword. proof

This statement is not scoped and any variables defined in the statement are still present outside in their parent scope no matter the depth of their nesting.

if true a = 90 print('hello world') print(a) // 90

If one of the statements phases executes, the others won't bother. Skipping to the next code block, execution starts from top to bottom.

keyword usage
if and elif followed by a conditional statement then demacated code
else followed by its demacted code.
pwd='123' user = prompt(': ') if (user == pwd) print('logged in') elif (user == '321') print('temp log in') else print('Wrong password')

If one executes the others won't bother and the execution starts from top to bottom.

Calling break and return from here is very okay since there is no new scope formed.

While loop

Similar to the if statement, no new scope is formed for loops. This while loop runs as long as the supplied condition statement is true or if the break is called.

Variables formed here are found in the parent scope too and we recommend not implementing any scoped state just to keep a safe habit.

// will never stop while (true){ print('hello world!') }
count = 10000 while count { print(count) count = count-1 } print('done')

The break keyword is very useful with its sister continue that is not implimented yet in the build, that will be fine soon.

count = 100 // ends at 77 while count { print(count) if(count == 77) break count = count-1 }

For loop

This is like the while loop just with a fixed incrementing or decrementing statement. Its fit for counts, any loop involving numbers.

And also here break works very fine.

fruits = ['apple', 'banana', 'pear', 'cherry'] for(let i=0; i < len(fruits); i++) print(fruits[i])

How it works?
First you let a random variable lets say i. Follow with an inequality statement involving i. Then use ++ to increment or -- to decrement i. In the start i can be assigned to any integer as its starting value but defaults to 0.

Classes

This is a special case as luqix classes are used to create your own datatypes call them objects. They are a cool building block.

You give them a name then functions and variables in there scope all become there attributes, this includes the __init__ function self variables.

class Student{ fn __init__(name, age){ self.name = name self.age = age } fn hey(){ print(f'My name is {self.name} and am {self.age} years old') } }

We have created a class or type Student thats kept in our scope for object duplication. So we call the type like a function to create an instant of the type. You supply arguments which will be for the __init__ function if present.

stream = [Student('Alice', 18), Student('Joan', 17)] stand = Student('Lin', 20) print(stand.age, stand.age) // Lin 20

You can create as many of these as possible, thats what makes luqix object oriented.

From and import statements

These two are used to utilize code thats broken into different files. This is very handy interms of big projects or distributing code coming soon.

import statement

import is followed by filenames without the extension .lqs.

import calculations import trigonometry as trig // folder/file import maths.statistics as stat

All these files must be in paths in the sys.path list, which you can update like append to from any where in your code.

sys = require('sys') print(sys.path)

Atleast the current working directory is already there

from statement

from is followed by a string (file or folder name) from the scope then an import statement.

from 'maths' import statistics check = 'maths' from check import trig
